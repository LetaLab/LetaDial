# LetaDial

**Personal speed dial dashboard — self-hosted, private, fast.**

A browser speed dial replacement you host yourself. Groups, thumbnails, 2FA, dark mode, import/export — no tracking, no ads, no cloud.

![MySQL](https://img.shields.io/badge/MySQL%2FMariaDB-10.6%2B-blue)

## Features

- Speed dial grid with custom thumbnails (auto-generated or uploaded)
- Groups with emoji icons, custom colors, custom image icons
- Full-text search, sort options (manual drag & drop, A→Z, popular, newest)
- Two-factor authentication (TOTP — Google Authenticator, Bitwarden, Authy)
- Import / export JSON
- Dark mode
- Keyboard navigation
- Dial notes with hover tooltip
- Bulk select, move, duplicate, delete
- Auto-fetch page title and description when adding a dial
- Settings page: password change, backup codes management
- Auto-update notifications from GitHub Releases
- Mobile-responsive

## Requirements

| Component | Minimum |
|---|---|
| PHP | 8.1+ |
| MySQL / MariaDB | 10.6+ |
| PHP extensions | `pdo_mysql`, `gd` (with WebP), `mbstring`, `openssl`, `json` |
| Web server | nginx or Apache with mod_rewrite |
| HTTPS | Strongly recommended |

Check requirements before installing:

```bash
php -r "
echo 'PHP: ' . PHP_VERSION . PHP_EOL;
echo 'PDO MySQL: '  . (extension_loaded('pdo_mysql') ? 'OK' : 'MISSING') . PHP_EOL;
echo 'GD: '         . (extension_loaded('gd') ? 'OK' : 'MISSING') . PHP_EOL;
echo 'WebP: '       . (!empty(gd_info()['WebP Support']) ? 'OK' : 'MISSING') . PHP_EOL;
echo 'mbstring: '   . (extension_loaded('mbstring') ? 'OK' : 'MISSING') . PHP_EOL;
echo 'OpenSSL: '    . (extension_loaded('openssl') ? 'OK' : 'MISSING') . PHP_EOL;
"
```

## Installation

### 1. Create database

```sql
CREATE DATABASE letadial_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'letadial_user'@'localhost' IDENTIFIED BY 'your_strong_password';
GRANT ALL PRIVILEGES ON letadial_db.* TO 'letadial_user'@'localhost';
FLUSH PRIVILEGES;
```

### 2. Upload files

Upload all files to your web root (e.g. `/var/www/html/dashboard/`).

```
/var/www/html/dashboard/
├── api/
├── assets/
├── install.php       ← installer
├── index.php
├── pages/
├── src/
└── storage/
```

### 3. Set permissions

```bash
chown -R www-data:www-data /var/www/html/dashboard/
find /var/www/html/dashboard/ -type d -exec chmod 755 {} \;
find /var/www/html/dashboard/ -type f -exec chmod 644 {} \;
```

### 4. Configure nginx

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    root /var/www/html/dashboard;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    location ~ /\. { deny all; }

    location ~* \.(ini|log|conf|bak|sql)$ { deny all; }

    location ~ \.php(?:$|/) {
        fastcgi_pass unix:/run/php-fpm/php-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param HTTPS on;
        include fastcgi_params;
    }
}
```

### 5. Run installer

Navigate to `https://your-domain.com/install.php` and follow the steps:

1. **Requirements** — verify all checks pass
2. **Database** — enter DB credentials
3. **Admin** — create admin account (login, email, password)
4. **Email** — optional SMTP for activation emails
5. **Done** — installer self-deletes, `config.php` created

### 6. Set up 2FA

Admin accounts **require** two-factor authentication. On first login you will be redirected to set up TOTP with any authenticator app.

## Directory structure

```
api/           HTTP endpoints (routed by index.php)
assets/        CSS, JS, icons
pages/         HTML page templates
src/           PHP classes
storage/       User data (thumbnails, sessions) — not web-accessible
install.php    Installer (auto-deletes after install)
index.php      Main router
config.php     Generated by installer — NEVER commit this file
```

## Auto-update notifications

LetaDial can check GitHub Releases for new versions and show a banner to admin users.

Add to `config.php` after installation:

```php
define('GITHUB_REPO',       'LetaLab/LetaDial');
define('UPDATER_CACHE_TTL', 21600); // 6 hours
```

Results are cached in the database — GitHub API is called at most once per TTL period.

## Security notes

- `config.php` contains database credentials and secret keys — **never commit it**
- `storage/` directory is protected by `.htaccess` — not web-accessible
- All API endpoints require authentication + CSRF tokens
- 2FA is mandatory for admin accounts
- Passwords: bcrypt cost 12, minimum 12 characters
- Sessions: DB-backed with SHA-256 token hashing
- Rate limiting on login, 2FA, imports, thumbnail generation
